ПРИНЦИПЫ РАЗРАБОТКИ В IDE

1. Что такое IDE?

IDE (Integrated Development Environment) — это "комбайн для программиста": всё в одном месте — редактор кода, запуск, отладка, управление файлами.

Аналогия:

IDE — это как кухня для повара:

  • стол = редактор кода

  • плита = запуск программы

  • холодильник = базы данных

  • таймер = отладка ошибок

IDE = готовая рабочая среда!

2. Что можно делать в любой IDE?

✅ Создавать проект ✅ Создавать папки и файлы ✅ Писать код с подсказками ✅ Запускать и тестировать программы ✅ Искать ошибки в коде ✅ Управлять библиотеками (через терминал) ✅ Работать с Git (управлять версиями проекта)

PyCharm (от JetBrains)

  • Специально для Python

  • Сам автоматически предлагает создать venv

  • Автоподстановка кода (супер умная)

  • Встроенный Git, Docker, базы данных

  • Минус: тяжеловат на слабых ПК

✅ Отлично подходит для больших проектов

VSCode (Visual Studio Code)

  • Лёгкий, быстрый

  • Требует установки расширений (например, Python Extension)

  • Поддержка кучи языков сразу

  • Встроенный терминал

  • Git интеграция

  • Очень удобен для Flask, FastAPI, скриптов

Thonny

  • Для новичков!

  • Всё просто: кнопка "Запустить", "Сохранить", и всё

  • Встроенный интерпретатор Python

  • Без заморочек с виртуальными окружениями

✅ Отлично подходит для изучения основ Python

  • Не для сайтов, а для данных, графиков, AI

  • Можно писать текст и код вперемешку

  • Красиво для отчётов, презентаций

✅ Отлично подходит для анализов данных, машинного обучения

Sublime Text, Atom

  • Лёгкие редакторы

  • Больше текстовые, чем IDE

✅ Хорошо для маленьких скриптов

НАСТРОЙКА ПРОЕКТА В PYCHARM И VSCODE

🔵 1. PyCharm — настройка проекта

ШАГИ:

1. Открываем PyCharm

  • Нажимаем File → New Project или Open (если папка уже есть)

2. Создаём виртуальное окружение сразу

  • При создании проекта PyCharm сам предложит:

    • ❓ New environment using venv

    • ✅ Нажимаешь Да!

    • Выбираешь путь к Python (python.exe)

(Или потом через: Settings → Project → Python Interpreter → Add → New venv)

4. Настройка запуска

  • Нажми Edit Configurations (вверху возле зелёной кнопки ▶️)

  • Нажми ➕ Python

  • Путь до файла: app/main.py

  • Имя: например Запуск проекта

Теперь нажимаешь зелёную кнопку ▶️ — и проект стартует!

ПРАВИЛЬНАЯ СТРУКТУРА ПРОЕКТА PYTHON

1. Базовая структура проекта

bash
проект/
├── venv/                # виртуальное окружение (НЕ трогай руками)
├── app/                 # основное приложение
│   ├── __init__.py      # делает папку модулем
│   ├── main.py          # основной файл запуска
│   ├── models.py        # классы моделей (если нужны)
│   └── utils.py         # вспомогательные функции
├── requirements.txt     # список библиотек
├── README.md            # описание проекта
└── .gitignore           # что НЕ заливать на GitHub

2. Пояснения: что где лежит

Папка/файлЧто это
venv/твоя песочница, библиотеки проекта
app/весь твой код сюда: функции, классы
main.pyточка входа в программу (отсюда запускаем)
requirements.txtкакие библиотеки нужны
README.mdобъяснение: что это за проект и как им пользоваться
.gitignoreчтобы случайно не отправить venv в GitHub

3. Как правильно создавать

✅ Открываешь терминал в папке проекта ✅ Делаешь:

bash
python -m venv venv    # создать виртуальное окружение
venv\Scripts\activate   # активировать venv
pip install нужные_библиотеки
pip freeze > requirements.txt  # сохранить список

✅ Создаёшь папку app/, туда пишешь свой код.

✅ В main.py подключаешь свой код из app/.

1. Простое объяснение:

venv = виртуальное окружение. Это такая песочница для твоего проекта, где ставятся библиотеки только для этого проекта, а не для всего компьютера.

Ты строишь кухню для себя. На ней свои продукты, свои кастрюли, и никто больше туда не лезет! 🍳 В другом проекте — другая кухня.

2. Зачем нужен venv?

✅ Чтобы библиотеки разных проектов не мешали друг другу ✅ Чтобы было легко удалить проект без грязи ✅ Чтобы на другом компьютере точно воспроизвести проект (через requirements.txt)

1. Что такое блочная разметка?

В Python структура кода определяется не скобками {}, как в C++/Java, а отступами (пробелами или табуляцией).

Отступы = показывают, что принадлежит какому блоку кода.

Аналогия:

Как лесенка 👣: То, что чуть правее, принадлежит родителю сверху.

2. Почему это важно?

✅ Python понимает: где начинается и где заканчивается тело функции, цикла, условия. ✅ Без правильной разметки код сломается! (Ошибка IndentationError)

3. Пример правильной блочной разметки

python
if5 > 3:
    print("Пять больше трёх!")  # Отступ 4 пробелаprint("Это внутри условия.")

print("Это вне условия.")  # Без отступа

Всё, что под if, — с отступом!

4. Сколько пробелов?

  • Стандарт: 4 пробела для одного уровня.

  • Не смешивай табы и пробелы! (Будет ошибка.)

ПЕРЕМЕННЫЕ В PYTHON

Что такое переменная?

Переменная — это коробка с названием. Ты кладёшь в неё что-то (например, число, слово, список), и потом можешь это использовать, просто назвав коробку.

1. Представь коробку и стикер

Пример:

python
яблоки = 3

Перевод на человеческий:

  • яблоки — это название коробки.

  • 3 — это то, что ты положил внутрь.

  • = — это как "положить в коробку".

Теперь ты можешь в любой момент сказать:

python
print(яблоки)

И Python ответит: 3

2. Что можно положить в переменную?

Примеры:

python
имя = "Аня"# строка — текст, пишем в кавычках
возраст = 25# число — без кавычек
рост = 1.65# дробное число
влюблён = True# логическое значение: True (да) или False (нет)

Ты можешь использовать переменные как математические значения:

python
a = 10
b = 5
результат = a + b
print(результат)   # Выведет 15

4. Переменные можно менять!

python
настроение = "грусть"
настроение = "радость"print(настроение)  # Выведет "радость"

Важно: переменная не навсегда = можно переписать, как будто ты заменил содержимое коробки.

5. Правила имён переменных

  • Нельзя начинать с цифры: 1яблоко — нельзя, яблоко1 — можно.

  • Нельзя использовать пробелы: мое имя — нельзя, мое_имя — можно.

  • Лучше писать по смыслу: скорость, вес_в_кг, результат_теста и т.д.

Переменная = ярлык на коробке, внутри которой лежит что-то нужное. Ты можешь это использовать, изменять, складывать, выводить на экран.

ЦИКЛ WHILE В PYTHON

Что такое while?

while — это повторение действий, пока условие ИСТИННО.

На русском:

Пока что-то происходит — продолжай делать это.

1. Структура цикла while

python

2. Простой пример: счёт от 1 до 5

python

    print("Счёт:", счёт)
    счёт = счёт + 1

Что делает Python:

  • Сначала проверяет: счёт <= 5?

  • Если да — выполняет тело цикла.

  • Потом снова проверяет.

  • И так по кругу, пока условие true.

Счёт: 1Счёт: 2Счёт: 3Счёт: 4Счёт: 5

3. Важное правило — остановка!

Если не остановишь — будет БЕСКОНЕЧНЫЙ ЦИКЛ. Пример неправильный:

python
whileTrue:
    print("Я застрял навсегда!")

Он будет печатать бесконечно, пока ты сам не выключишь программу.

Пространство имён — это список имён, которые Python знает в данный момент.

Или проще:

  4. Built-in → встроенное

Пример: Ты делишь на ноль → программа падает

python
print(10 / 0)  # ❌ Ошибка: деление на ноль!

Обработка исключений — это как ловушка для ошибок.

Если ошибка произойдёт — мы её поймаем, не дадим программе упасть, и спокойно отреагируем.

1. Синтаксис try-except

python
try:
    # тут может быть ошибкаexcept:
    # что делать, если она случилась

2. Пример: деление с защитой

python
try:
    x = int(input("Введи число: "))
    print(10 / x)
except:
    print("Ошибка! Делить на ноль или вводить буквы нельзя.")

3. Обработка конкретной ошибки

python
try:
    x = int("три")
except ValueError:
    print("Нельзя преобразовать в число!")

Популярные ошибки:

МНОЖЕСТВА

1. Что такое множество (set)?

Множество — это набор уникальных элементов. В отличие от списка или кортежа, в множество нельзя добавить одинаковые элементы.

Пример: Если ты создашь множество с элементами, которые повторяются, то в результате они исчезнут.

python
fruits = {"яблоко", "банан", "яблоко", "груша"}
print(fruits)  # {'яблоко', 'банан', 'груша'}

2. Для чего нужны множества? 🔍

Множества полезны, когда нужно:

  • Удалить дубликаты из списка.

  • Проверить уникальность элементов.

  • Работать с математическими операциями, такими как пересечение, объединение и разность.

3. Как создать множество? 🛠️

Создание множества:

python
fruits = {"яблоко", "банан", "груша"}

Пустое множество:

python
empty_set = set()

Важно! Пустой контейнер с фигурными скобками {} — это не пустое множество, а пустой словарь. Для пустого множества нужно использовать set().

4. Операции с множествами 🔄

Добавление элемента в множество

python
fruits = {"яблоко", "банан", "груша"}
fruits.add("апельсин")
print(fruits)  # {'яблоко', 'банан', 'груша', 'апельсин'}

Удаление элемента

  • .remove() — удаляет элемент, но если его нет, вызывает ошибку.

python
fruits.remove("банан")  # удалили "банан"print(fruits)  # {'яблоко', 'груша', 'апельсин'}

  • .discard() — удаляет элемент, но не вызывает ошибку, если его нет.

python
fruits.discard("манго")  # ничего не произойдётprint(fruits)  # {'яблоко', 'груша', 'апельсин'}

  • .pop() — удаляет и возвращает случайный элемент.

python
fruit = fruits.pop()
print(fruit)  # например, "яблоко"

Очистить множество

python
fruits.clear()  # очистить всё множествоprint(fruits)  # set()

5. Математические операции с множествами ➕➖

Объединение

Объединяет два множества (сохраняются только уникальные элементы).

python
a = {1, 2, 3}
b = {3, 4, 5}
result = a | b  # объединениеprint(result)  # {1, 2, 3, 4, 5}

Пересечение

Возвращает элементы, которые есть и в одном, и в другом множестве.

python
result = a & b  # пересечениеprint(result)  # {3}

Разность

Возвращает элементы, которые есть в одном множестве, но нет в другом.

python
result = a - b  # разностьprint(result)  # {1, 2}

Симметрическая разность

Возвращает элементы, которые есть только в одном из множеств, но не в обоих.

python
result = a ^ b  # симметрическая разностьprint(result)  # {1, 2, 4, 5}

6. Проверка подмножеств и надмножеств

Подмножество

Множество a является подмножеством множества b, если все элементы a есть в b.

python
a = {1, 2}
b = {1, 2, 3, 4}

print(a.issubset(b))  # True

Надмножество

Множество a является надмножеством множества b, если все элементы b есть в a.

python
a = {1, 2, 3, 4}
b = {1, 2}

print(a.issuperset(b))  # True

7. Замороженное множество (frozenset) 🔒

Что это такое?

  • frozenset — это неизменяемая версия множества.

  • Нельзя добавить или удалить элементы из frozenset после его создания.

Зачем нужно?

  • Полезно, когда нужно использовать множество в качестве ключа в словаре или элемента множества.

Как создать frozenset?

python
frozen_fruits = frozenset(["яблоко", "банан", "груша"])
print(frozen_fruits)  # frozenset({'яблоко', 'банан', 'груша'})

Важно! Как только создаёшь frozenset, ты уже не можешь добавить или удалить элементы.

8. Сравнение множеств ⚖️

Множества можно сравнивать между собой для проверки равенства или содержания:

python
a = {1, 2, 3}
b = {1, 2, 3}
c = {4, 5, 6}

print(a == b)  # True (равны)print(a != c)  # True (не равны)

БИНАРНЫЕ ФАЙЛЫ В PYTHON

1. Что такое бинарный файл?

Бинарный файл — это файл, где данные хранятся не как текст, а в байтах (единицах и нолях).

Примеры бинарных файлов:

  • Картинки (.jpg, .png)

  • Видео (.mp4)

  • Звук (.mp3)

  • Программы (.exe)

  • Документы Word, PDF и др.

Ты не можешь их прочитать обычным способом ("r") — будет мусор.

2. Зачем использовать бинарные файлы?

  • Чтобы сохранять и читать изображения, музыку, файлы, которые не текст.

  • Чтобы работать с байтами напрямую (например, при передаче по сети, шифровании и т.д.)

3. Как открыть бинарный файл

Чтение (read) — 'rb'

python
withopen("фото.jpg", "rb") as f:
    байты = f.read()
    print(байты[:10])  # первые 10 байтов

'rb' = read binary

Запись (write) — 'wb'

python
withopen("копия.jpg", "wb") as f:
    f.write(байты)  # записали байты из предыдущего примера

'wb' = write binary

Копирование бинарного файла:

python
withopen("фото.jpg", "rb") as исходник:
    содержимое = исходник.read()

withopen("фото_копия.jpg", "wb") as копия:
    копия.write(содержимое)

4. Как это работает в памяти?

Когда ты читаешь файл в "rb" — ты получаешь байтовый объект (bytes), а не строку:

python
b = b"Привет"# это байтовая строка, не обычная

5. Частичное чтение бинарного файла (по кусочкам)

python

    whileTrue:
        chunk = f.read(1024)  # читаем по 1024 байтаifnot chunk:
            break# тут можно что-то делать с chunk

Это удобно, если файл ОЧЕНЬ большой и нельзя загрузить весь в память.

6. Сериализация (сохранение Python-объектов в бинарный файл)

1. Что такое конструктор?

Конструктор — это спец-функция, которая вызывается при создании объекта. В Python он называется __init__().

Аналогия:

Класс — это форма для печенья Конструктор — это первый шаг, который формирует печеньку: Он заполняет объект начальными данными.

2. Зачем нужен конструктор?

  • Чтобы автоматически задавать значения, когда создаёшь объект.

  • Чтобы ничего не забыть и всё сразу настроить.

  • self — это сам объект

  • Всё, что ты передал в скобках ("Петя", 30) — попадает в конструктор

ИНКАПСУЛЯЦИЯ

1. Что такое инкапсуляция?

Инкапсуляция — это защита данных внутри объекта. То есть скрытие внутренней “кухни” объекта, чтобы никто случайно не сломал его изнутри.

2. Зачем нужна?

  • Чтобы скрыть “внутренности” объекта (переменные, расчёты)

  • Чтобы никто не вмешивался напрямую

  • Чтобы данные менялись через безопасные методы

Аналогия:

Кофемашина — Ты нажимаешь кнопку "капучино", и получаешь напиток. — Ты не лезешь внутрь, не мешаешь молоко и кофе вручную.

4. Пример с инкапсуляцией

Всё, что начинается с __, Python переименовывает "внутри":

ПОЛИМОРФИЗМ

1. Что такое полиморфизм?

Полиморфизм — это когда разные объекты делают одно и то же, но по-своему.

Слово "поли" — много, "морфизм" — формы. → Одна команда, много форм поведения.

Аналогия:

Команда "ГОВОРИ" —

  • Собака скажет: Гав!

  • Кошка скажет: Мяу!

  • Попугай скажет: Привет!

→ Команда одна, поведение разное.

2. Зачем нужен полиморфизм?

  • Чтобы писать универсальный код, не думая о типе объекта.

  • Чтобы перебрать список разных объектов и вызвать у каждого "одинаковый метод", а они сами решают, как себя вести.

3. Пример на классах

python
classЖивотное:
    defговорить(self):
        print("Животное молчит")

classСобака(Животное):
    defговорить(self):
        print("Гав!")

classКошка(Животное):
    defговорить(self):
        print("Мяу")

4. Один метод — много форм

python
питомцы = [Собака(), Кошка(), Животное()]

for животное in питомцы:
    животное.говорить()

Результат:

Гав!
Мяу
Животное молчит

ГЕНЕРАЦИЯ ТЕКСТА С LSTM

1. Что такое LSTM?

LSTM — это вид нейросети (RNN), которая умеет запоминать последовательности: буквы, слова, музыку, шаги — всё, что идёт по порядку.

LSTM = Long Short-Term Memory — умеет помнить, что было раньше в тексте.

2. Что можно сделать с LSTM?

  • Сгенерировать текст в стиле Пушкина

  • Продолжить фразу

  • Написать статью, как робот

  • Сочинить код, песню, рецепт

3. Что нужно для генерации текста?

✅ Набор текста для обучения (например, книга) ✅ Разбивка на символы или слова ✅ Модель LSTM ✅ Обучение ✅ Генерация новых фраз

4. Простой пример на Keras (генерация по буквам)

python
import numpy as np
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense
from tensorflow.keras.utils import to_categorical

# 1. Исходный текст
текст = "мама мыла раму"# 2. Создаём словарь символов
буквы = sorted(set(текст))
char_to_int = {c: i for i, c inenumerate(буквы)}
int_to_char = {i: c for c, i in char_to_int.items()}

# 3. Подготовка последовательностей
seq_length = 3
dataX = []
dataY = []

for i inrange(len(текст) - seq_length):
    seq_in = текст[i:i + seq_length]
    seq_out = текст[i + seq_length]
    dataX.append([char_to_int[ч] for ч in seq_in])
    dataY.append(char_to_int[seq_out])

# 4. Преобразование в массивы
X = np.reshape(dataX, (len(dataX), seq_length, 1)) / len(буквы)
y = to_categorical(dataY)

# 5. Создание модели
model = Sequential()
model.add(LSTM(128, input_shape=(seq_length, 1)))
model.add(Dense(len(буквы), activation='softmax'))
model.compile(loss='categorical_crossentropy', optimizer='adam')

# 6. Обучение
model.fit(X, y, epochs=500, verbose=0)

# 7. Генерация текста
начальный_текст = dataX[0]  # например: мама → мамfor _ inrange(20):
    x = np.reshape(начальный_текст, (1, seq_length, 1)) / len(буквы)
    предсказание = model.predict(x, verbose=0)
    индекс = np.argmax(предсказание)
    символ = int_to_char[индекс]
    print(символ, end='')
    начальный_текст.append(индекс)
    начальный_текст = начальный_текст[1:]

СОКЕТЫ В PYTHON

1. Что такое сокеты?

Сокет — это как розетка для данных: он соединяет 2 программы (или компьютера) по сети. Сокеты позволяют отправлять и получать данные между клиентом и сервером.

Аналогия:

  • Сервер — это телефонный оператор (ждёт звонков)

  • Клиент — это ты, кто звонит

  • Сокет — это провод между вами

2. Подключение библиотеки

python
import socket

3. Сервер — слушает

python
import socket

сервер = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
сервер.bind(("localhost", 12345))  # адрес и порт
сервер.listen(1)
print("Сервер ждёт подключение...")

клиент, адрес = сервер.accept()
print("Подключился клиент:", адрес)

данные = клиент.recv(1024).decode()
print("Получено:", данные)

клиент.send("Привет от сервера!".encode())

клиент.close()
сервер.close()

4. Клиент — подключается и отправляет

python
import socket

клиент = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
клиент.connect(("localhost", 12345))

клиент.send("Привет, сервер!".encode())
ответ = клиент.recv(1024).decode()
print("Ответ сервера:", ответ)

клиент.close()

5. Как это работает:

ЭтапЧто происходит

bind()привязывает адрес/порт

accept()принимает клиента
recv()получает данные
send()отправляет данные
close()закрывает соединение

КРИПТОГРАФИЯ В PYTHON

1. Что такое криптография?

Криптография — это шифрование информации, чтобы её никто не понял, кроме получателя.

Примеры:

  • Хранение паролей

  • Шифровка сообщений

  • Защита данных при передаче

2. Виды шифрования:

ТипЧто делает
Симметричноеодин и тот же ключ — для шифрования и дешифровки
Асимметричноедва ключа: открытый (public) и закрытый (private)
Хешированиеодностороннее — нельзя расшифровать (только проверить совпадение)

4. Симметричное шифрование (Fernet)

from cryptography.fernet import Fernet

# 1. Создаём ключ

cipher = Fernet(ключ)

# 2. Шифруем

# 3. Расшифровываем

5. Хеширование (одностороннее)

Хеш нельзя расшифровать! Только сравнивать, совпадает ли с оригиналом.

6. Асимметричное шифрование (RSA)

python
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.hazmat.primitives import serialization, hashes
from cryptography.hazmat.primitives.asymmetric import padding

# Генерация ключей
private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
public_key = private_key.public_key()

# Шифрование (открытым ключом)
сообщение = b"секретное сообщение"
зашифровано = public_key.encrypt(
    сообщение,
    padding.OAEP(mgf=padding.MGF1(algorithm=hashes.SHA256()), algorithm=hashes.SHA256(), label=None)
)

# Расшифровка (приватным ключом)
расшифровано = private_key.decrypt(
    зашифровано,
    padding.OAEP(mgf=padding.MGF1(algorithm=hashes.SHA256()), algorithm=hashes.SHA256(), label=None)
)

print("Расшифровано:", расшифровано.decode())

МОКИРОВАНИЕ (MOCKING)

1. Что такое мок (mock)?

Mock — это поддельный (фейковый) объект, который ведёт себя как настоящий, но без настоящей логики. Он нужен, чтобы тестировать код в изоляции, не вызывая настоящие базы данных, API, файлы и т.д.

Аналогия:

У тебя есть функция, которая вызывает банк для проверки баланса. Во время тестов ты не хочешь звонить в реальный банк — ты подменяешь банк на фейковый, и тестируешь поведение твоей функции без настоящего сервера.

2. Когда нужен mock?

  • Когда функция обращается к интернету, файлу, базе

  • Когда настоящие вызовы медленные, нестабильные или опасные

  • Когда ты хочешь проверить, был ли вызов и с какими аргументами

АСИНХРОННОЕ ПРОГРАММИРОВАНИЕ

1. Что это такое?

Асинхронность = не жди — делай дальше, пока что-то другое выполняется.

Python делает паузы в одном месте, чтобы не тупить, а продолжать выполнять другое.

Аналогия:

Ты готовишь чай. В обычном коде — стоишь и ждёшь, пока закипит. В асинхронном — включаешь чайник и... идёшь жарить яичницу! (Когда чайник вскипит — вернёшься.)

2. Где это нужно?

  • Работа с интернетом (API, Telegram-боты)

  • Ожидание ответа от базы данных

  • Задержки (sleep)

  • Большое количество одновременных задач без тяжёлых вычислений

3. Основы синтаксиса: async + await

Асинхронная функция

python
import asyncio

asyncdefсказать_привет():
    print("Привет")
    await asyncio.sleep(1)
    print("Прошла 1 секунда")

Запуск (Python 3.7+)

python
asyncio.run(сказать_привет())

4. Параллельные задачи

python
asyncdefзадача_1():
    await asyncio.sleep(2)
    print("Задача 1 готова")

asyncdefзадача_2():
    await asyncio.sleep(1)
    print("Задача 2 готова")

asyncdefmain():
    await asyncio.gather(
        задача_1(),
        задача_2()
    )

asyncio.run(main())

Задача 1 готова

  • Говорит: "подожди тут, а пока делай что-то другое"

  • Работает только внутри async def

  • Только с асинхронными функциями

CELERY — ОТЛОЖЕННЫЕ ЗАДАЧИ В PYTHON

1. Что такое Celery?

Celery — это инструмент для запуска задач в фоне. Ты говоришь: “Сделай это позже” → Celery делает это в другом потоке, процессе или даже на другом сервере.

Аналогия:

У тебя пекарня. К тебе приходит клиент и говорит: "Испеки пирог". Вместо того чтобы стоять и печь 2 часа — ты кидаешь заказ в "очередь задач", а Celery — это как фоновый повар, который печёт, пока ты обслуживаешь других клиентов.

2. Где используется Celery?

  • Отправка email

  • Генерация отчётов

  • Фоновый импорт данных

  • Медиа-обработка (например, видео, PDF)

  • Задержка: "через 10 минут"

3. Что нужно?

✅ Python ✅ Celery ✅ Брокер (очередь задач): чаще всего Redis или RabbitMQ

4. Установка

bash
pip install celery redis

И Redis:

bash
sudo apt install redis-server

(или запусти через Docker)

5. Простейший пример

Файл: tasks.py

python
from celery import Celery

app = Celery('my_app', broker='redis://localhost:6379/0')

@app.taskdefприбавить(x, y):
    return x + y

6. Запуск воркера (исполнителя задач)

bash
celery -A tasks worker --loglevel=info

7. Отправка задачи из Python

python
from tasks import прибавить

результат = прибавить.delay(4, 6)
print(результат.get())  # 10

.delay() — ставит задачу в очередь .get() — ждёт результат

8. Планирование задач (по времени)

Можно подключить Celery Beat:

bash
pip install celery[redis] django-celery-beat

Пример: запускать задачу каждые 10 секунд.

9. Интеграция с Flask или Django

Celery отлично встраивается в:

  • Django (через celery.py)

  • Flask (через расширение или вручную)

Иконки-шпаргалка:

ИконкаЧто
🧾.task — помечает задачу
📦.delay() — ставит в очередь
🧠worker — выполняет задачи
⏱beat — планировщик
🔄Redis — очередь между ними

Вывод:

ЧтоКак
Создать задачу@app.task
Отправитьзадача.delay(...)
Получить результат.get()
ОчередьRedis или RabbitMQ
Воркерыcelery -A имя worker

5. Простейшая нейросеть

python
import torch.nn as nn
import torch.optim as optim

# 1. Данные (пример)
x = torch.tensor([[1.0], [2.0], [3.0]])
y = torch.tensor([[2.0], [4.0], [6.0]])

# 2. Модель (один слой)
model = nn.Linear(1, 1)

# 3. Потери и оптимизация
criterion = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.01)

# 4. Обучениеfor epoch inrange(100):
    y_pred = model(x)
    loss = criterion(y_pred, y)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()

print(model(torch.tensor([[4.0]])))  # предсказание

6. Что здесь происходит?

ЭтапЧто делает

criterionфункция ошибки (MSE)
optimizerобновляет веса
backward()рассчитывает градиенты
step()делает шаг обучения

  • services/ обрабатывают логику

  • models/ описывают данные

  • database/ подключает базу

  • utils/ помогают в мелочах

📋 Мини-инструкция по развёртке такого проекта

  1. Создать venv и активировать его

  2. Установить библиотеки:

bash
pip install fastapi uvicorn sqlalchemy pydantic
pip freeze > requirements.txt

  1. Стартануть проект:

bash
uvicorn app.main:app --reload

  1. Перейти в браузере на:

  • http://127.0.0.1:8000/docs → Swagger-документация
